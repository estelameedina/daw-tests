¿Qué característica distintiva ofrece TypeScript, un superconjunto de JavaScript, que lo ha hecho ganar popularidad especialmente en proyectos de gran escala?
		 Es una librería ligera de JavaScript que facilita principalmente la manipulación del DOM y la gestión de eventos.   
		 Ofrece características de tipado estático y orientación a objetos, lo que facilita la escritura de código más limpio, robusto y mantenible.  
		 Permite la ejecución de código de bajo nivel en el navegador sin necesidad de plugins, con un rendimiento cercano al nativo.   
¿Por qué es fundamental una estructura HTML bien organizada al integrar el código con las etiquetas HTML en un proyecto web?
		 Solo es relevante para asegurar una correcta visualización en navegadores muy antiguos que no soportan CSS moderno.   
		 Mejora la legibilidad del código, facilita su mantenimiento a largo plazo y permite la escalabilidad del sitio web.  
		 Es el único método para proteger el sitio web de ataques externos, ya que no se necesitan protocolos de ataques externos, ya que no se necesitan protocolos de seguridad adicionales.   
Para optimizar el desarrollo y la ejecución de aplicaciones web en el entorno cliente, ¿qué configuración o práctica relacionada con el navegador es crucial?

		 Mantener el navegador constantemente actualizado y hacer uso de sus herramientas de desarrollo integradas (DevTools) para depuración y análisis.  
		 Ignorar las variaciones de rendimiento entre diferentes navegadores, ya que todos ejecutan el código de la misma manera.

		 Traer dinamismo e interactividad a las páginas web, permitiendo la manipulación del DOM y la respuesta a eventos del usuario en tiempo real.

¿Cuál es la principal diferencia en el alcance (scope) entre var y let/const en JavaScript?
		 VAR tiene alcance de bloque, mientras que let y const tienen alcance de función.   
		 VAR declara variables con alcance de función, mientras que let y const declaran variables con alcance de bloque, ofreciendo mayor control y reduciendo errores.  
		 LET y CONST son para declarar constantes, mientras que var es para variables modificables.

		 Asignar propiedades de un objeto o elementos de un arreglo a variables individuales de manera legible.  
¿Qué implica que JavaScript sea un lenguaje de tipado dinámico?
		 Requiere que el programador defina explícitamente el tipo de dato de una variable al declararla.   
		 Las variables pueden contener diferentes tipos de datos a lo largo de su vida útil, y su tipo se determina en tiempo de ejecución.  
		 No permite la reasignación de valores a las variables una vez que han sido declaradas con un tipo inicial.

		 Declarar todas las variables globalmente para asegurar su accesibilidad en cualquier parte del código.   
		 Reasignar constantemente el tipo de dato de una variable para demostrar la flexibilidad del tipado dinámico.   
		 Declarar variables en el ámbito más restringido posible y mantener la coherencia en los tipos de datos para evitar errores.

¿Cuál es la principal diferencia entre los operadores de comparación == (igualdad suelta) y === (igualdad estricta) en JavaScript?
		 No existe diferencia; ambos operadores realizan exactamente la misma comparación.   
		 El operador == realiza una conversión de tipos al comparar valores, mientras que === compara tanto el valor como el tipo de datos sin convertirlos.  
		 El operador == se utiliza únicamente para comparar valores numéricos, mientras que === se usa para comparar cadenas de texto y otros tipos de datos.
