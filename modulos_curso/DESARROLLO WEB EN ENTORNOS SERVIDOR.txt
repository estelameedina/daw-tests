Respecto al concepto de ámbito (scope) de las funciones, ¿cuál es la diferencia clave entre una variable global y una variable local?
		 Las variables globales se definen dentro de una función, mientras que las variables locales se definen fuera de cualquier función.   
		 Las variables locales persisten durante toda la ejecución del programa, a diferencia de las globales.   
		 Las variables globales son accesibles desde cualquier parte del código, mientras que las variables locales están limitadas a ser accesibles únicamente dentro de la función o bloque donde se definen.

		 El mantenimiento del estado es una optimización de rendimiento que reduce el número de solicitudes al servidor.

¿Cuál es una de las ventajas clave de utilizar PreparedStatement en Java para ejecutar consultas SQL?

		 Mejora la seguridad al prevenir ataques de inyección de SQL y optimiza el rendimiento al permitir la reutilización de consultas compiladas.  
		 Es la única forma de manipular un ResultSet actualizable directamente a través de JDBC.

		 Commit, para asegurar que las operaciones se apliquen de forma permanente, independientemente del error.   
		 Rollback, para deshacer todas las operaciones realizadas durante la transacción.

En el contexto de Java Database Connectivity (JDBC), ¿qué representa un ResultSet y cómo se obtiene principalmente?
		 Es un objeto que encapsula la conexión activa a la base de datos y se obtiene mediante DriverManager.getConnection().   
		 Es un conjunto de sentencias SQL precompiladas que se obtiene de Connection.prepareStatement() para mejorar el rendimiento.   
		 Es una estructura de datos que almacena el resultado obtenido de una consulta SQL de tipo SELECT, y se obtiene al ejecutar dicha consulta.  
Al ejecutar sentencias UPDATE o DELETE en una base de datos, ¿qué precaución es crítica para evitar cambios masivos o pérdidas de datos no deseadas?
		 Usar sentencias preparadas para que el motor valide automáticamente todas las entradas.   
		 Realizar un COMMIT tras cada operación para evitar inconsistencias.   
		 Incluir correctamente la cláusula WHERE para limitar los registros afectados.  
Para prevenir eficazmente los ataques de inyección de SQL, una de las medidas más importantes y recomendadas es:
		 Validar exhaustivamente todas las entradas del usuario únicamente en el lado del cliente (navegador).   
		 Utilizar sentencias preparadas, ya que separan los datos de las instrucciones SQL, evitando que las entradas malintencionadas se interpreten como código.  
		 Configurar el servidor de base de datos para que rechace cualquier consulta que contenga el carácter de comilla simple (').   
Dentro de las propiedades ACID que rigen las transacciones en bases de datos, ¿a qué se refiere la "Atomicidad" (Atomicity)?
		 Asegura que, una vez que una transacción se ha completado, sus efectos son permanentes en la base de datos.   
		 Determina cómo la ejecución de una transacción puede ser visible para otras transacciones, previniendo interferencias.   
		 Asegura que todas las operaciones dentro de la transacción se realizan completamente o ninguna; si alguna parte falla, toda la transacción se revierte.

		 Acceso directo a archivos locales (CSV, XML, JSON) almacenados en el servidor web.   
		 El uso de APIs (Application Programming Interfaces) y servicios web, como las APIs RESTful.  
		 La integración mediante plugins o módulos específicos proporcionados por un CMS (Sistema de Gestión de Contenido).   
¿Cuál es el propósito de la serialización en el desarrollo web?
		 Proteger la información mediante cifrado automático antes de enviarla al cliente.   
		 Mejorar el rendimiento de la base de datos almacenando consultas en caché.   
		 Transformar objetos en formatos como JSON o XML para almacenarlos o transmitirlos, y reconstruirlos después.  
Un riesgo de seguridad significativo asociado con la serialización, si no se maneja con cuidado, es:
		 El aumento del consumo de ancho de banda debido al mayor tamaño de los datos serializados.   
		 La dificultad de depurar el código cuando los objetos están serializados.   
		 Las vulnerabilidades como ataques de inyección de código que pueden surgir al deserializar datos no confiables o maliciosos.

¿Cuál es una diferencia fundamental entre SOAP y REST?
		 SOAP utiliza siempre JSON, mientras que REST solo permite XML.   
		 SOAP intercambia mensajes en XML con un estándar estricto, mientras que REST se basa en HTTP y suele usar formatos más ligeros como JSON.  
		 SOAP es más simple y eficiente que REST en la mayoría de los casos.   
Al implementar las operaciones básicas de un servicio web, ¿qué es crucial al definir y diseñar los "Endpoints"?
		 Asegurarse de que todos los endpoints utilicen únicamente el método HTTP GET para simplificar la interfaz.   
		 Priorizar la implementación de la lógica de negocio antes de definir la estructura de la URL y los parámetros.   
		 Definir claramente la estructura de la URL, los parámetros esperados y el formato de respuesta para cada operación, asociándolos con el método HTTP (GET, POST, PUT, DELETE) adecuado.  
Al implementar operaciones básicas que requieren interacción con una base de datos, ¿qué herramienta o técnica se menciona como una forma de facilitar esta interacción al permitir trabajar con objetos en el código que se mapean a las tablas de la base de datos?
		 La creación manual de conexiones individuales y consultas SQL para cada operación para maximizar el control.   
		 La configuración de un pool de conexiones para asegurar la integridad de los datos en transacciones complejas.   
		 El uso de un ORM (Object-Relational Mapping) para simplificar el desarrollo y mantener el código más limpio.  
En cuanto a la seguridad, ¿cuál es una diferencia clave entre XML-RPC y SOAP?
		 Ambos protocolos tienen mecanismos de seguridad integrados, siendo XML-RPC el más robusto gracias a sus especificaciones avanzadas.   
		 SOAP no tiene un mecanismo de seguridad integrado y debe usar HTTPS, mientras que XML-RPC puede integrar seguridad a nivel de mensaje mediante WS-Security.   
		 XML-RPC no tiene un mecanismo de seguridad integrado y su seguridad debe manejarse a través de otros medios como HTTPS, a diferencia de SOAP que puede integrar seguridad a nivel de mensaje mediante WS-Security.  
Además de la encriptación mediante HTTPS, ¿qué mecanismo se utiliza comúnmente en los servicios RESTful para verificar la identidad de los usuarios y controlar el acceso a los recursos del servicio web?
		 El estándar WS-Security para proporcionar mecanismos de autenticación a nivel de mensaje.   
		 La validación estricta de las entradas del usuario solo en el lado del cliente (navegador).   
		 Los tokens de autenticación, como JWT (JSON Web Tokens), para la verificación de la identidad y el control de acceso.  
¿Cuál es una ventaja clave que hace que JSON sea ampliamente utilizado en los servicios web modernos, especialmente en los servicios RESTful, en comparación con XML?
		 JSON ofrece una mayor capacidad para definir esquemas y estructuras de datos complejas de forma detallada.   
		 JSON es el formato preferido para servicios basados en SOAP debido a su naturaleza auto-descriptiva.   
		 JSON es un formato ligero, más fácil de leer y escribir para los humanos y más eficiente de analizar y generar para las máquinas, lo que optimiza la transmisión de datos.  
¿Por qué se considera la documentación de la API un componente crítico en la descripción de un servicio web?
		 Su propósito principal es monitorear el rendimiento del servicio y registrar los errores en tiempo real.   
		 Porque es una guía técnica esencial que explica cómo interactuar con el servicio, detallando los endpoints, métodos, parámetros de entrada, formatos de respuesta y posibles códigos de error, facilitando su integración y uso efectivo.  
		 Su objetivo principal es aplicar políticas de seguridad y cumplimiento normativo de forma automática.   
¿Cuál es la función principal de un documento WSDL (Web Services Description Language) en el contexto de los servicios web?
		 Su principal función es cifrar los mensajes de seguridad en los servicios web para proteger los datos sensibles.   
		 Es un protocolo que facilita la comunicación y el intercambio de datos entre sistemas a través de Internet, utilizando XML para codificar llamadas y HTTP para el transporte.   
		 Es un formato basado en XML que describe cómo se pueden invocar los servicios web, especificando qué parámetros requieren y qué datos devuelven, sirviendo como un contrato formal de la interfaz del servicio.

¿Cuál es una distinción fundamental entre las Bases de Datos Relacionales (como MySQL) y las Bases de Datos NoSQL (como MongoDB)?
		 Las bases de datos relacionales están diseñadas para análisis de datos en tiempo real, mientras que las NoSQL son para el registro tradicional de datos.   
		 Las bases de datos relacionales utilizan un esquema fijo y son ideales para datos que requieren integridad referencial y transacciones complejas, mientras que las bases de datos NoSQL ofrecen flexibilidad de esquema y son adecuadas para grandes volúmenes de datos y escalabilidad horizontal.  
		 Las bases de datos NoSQL priorizan la consistencia estricta de los datos sobre la disponibilidad, a diferencia de las relacionales.

		 El uso exclusivo de archivos HTML estáticos para la entrega de contenido.

		 Permitir que partes de una página web se actualicen de manera asincrónica, intercambiando datos con el servidor sin necesidad de recargar la página completa.

Comparando RSS y Atom, ¿cuál de las siguientes afirmaciones describe correctamente una diferencia clave entre ellos?
		 RSS fue diseñado para ser más extensible y poderoso en cuanto a capacidades de metadatos que Atom.   
		 Atom ofrece una estructura más flexible y robusta, con mejor soporte para metadatos y la capacidad de soportar actualizaciones de contenido en el mismo feed, a diferencia de RSS.  
		 RSS es el estándar más moderno y con capacidades más avanzadas para el intercambio de datos en APIs web.
